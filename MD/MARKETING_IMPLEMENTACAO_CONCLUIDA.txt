⚡ QUICK START (3 PASSOS)
===========================

PASSO 1: Criar arquivo .env
────────────────────────────
Crie um arquivo "c:\crm-server\.env" com:

Para Gmail (Recomendado):
========================
SMTP_HOST=smtp.gmail.com
SMTP_PORT=587
SMTP_USER=seu-email@gmail.com
SMTP_PASS=sua-senha-de-app-16-caracteres


Para Outlook:
=============
SMTP_HOST=smtp-mail.outlook.com
SMTP_PORT=587
SMTP_USER=seu-email@hotmail.com
SMTP_PASS=sua-senha-outlook


PASSO 2: Obter Senha de App Gmail
──────────────────────────────────
Se usar Gmail:
1. Abra: https://myaccount.google.com
2. Clique "Segurança"
3. Procure "Senhas de app"
4. Selecione: Mail + Windows
5. Copie a senha gerada (16 caracteres)
6. Cole no .env


PASSO 3: Reiniciar Servidor
────────────────────────────
Parar: Ctrl+C
Rodar: npm start

Sucesso? Você verá:
✅ "Servidor de email configurado com sucesso"

💾 BANCO DE DADOS
===================

Automaticamente criadas 3 tabelas:

1. email_campanhas
   ├─ id: ID da campanha
   ├─ usuario_id: Quem enviou
   ├─ assunto: Assunto do email
   ├─ corpo: Conteúdo do email
   ├─ total_destinos: Quantos receberam
   ├─ total_enviados: Quantos foram entregues
   ├─ status: "enviado" / "pendente" / "erro"
   └─ data_criacao: Data do envio

2. email_logs
   ├─ id: ID do log
   ├─ campanha_id: FK para campaign
   ├─ cliente_id: FK para cliente
   ├─ email_destinatario: Email final
   ├─ nome_cliente: Nome do cliente
   ├─ status: "enviado" / "erro"
   ├─ erro_mensagem: Se houver falha
   └─ data_envio: Timestamp

3. Já existente (dados dos clientes)
   ├─ documents (colecção "clientes")
   └─ Emails lidos automaticamente

⚙️ VARIÁVEIS DE AMBIENTE (.env)
=================================

Necessárias:
SMTP_HOST      = Host do servidor SMTP
SMTP_PORT      = Porta (geralmente 587 ou 465)
SMTP_USER      = Email/usuário para autenticação
SMTP_PASS      = Senha ou token de autenticação

Exemplo:
SMTP_HOST=smtp.gmail.com
SMTP_PORT=587
SMTP_USER=meu-crm@gmail.com
SMTP_PASS=abcd efgh ijkl mnop

🔒 SEGURANÇA
=============

Protegido? SIM:

✅ Senha SMTP:
   ├─ Não fica no banco de dados
   ├─ Não é enviada ao frontend JavaScript
   ├─ Só existe no servidor Node.js
   ├─ Apenas em variáveis de ambiente

   ├─ Ficam locais (SQLite)

🐛 PROBLEMAS E SOLUÇÕES
==========================

Problema: "Email não configurado"
─────────────────────────────────
Causa: .env não criado ou não lido
Solução:
1. Crie arquivo .env
2. Adicione credenciais SMTP
3. Reinicie npm start
4. Mensagem deve mudar para ✅


Problema: "Erro ao enviar emails"
──────────────────────────────────
Causa provável: Credenciais incorretas
Solução:
1. Verifique dados no .env
2. Tente com senhas diferentes
3. Se Gmail, use senha de app (não senha normal)
4. Testar: npm start e ver logs


Problema: "Emails não chegam"
──────────────────────────────
Causa: Geralmente spam folder
Solução:
1. Peça ao cliente verificar SPAM
2. Teste com seu próprio email primeiro
3. Valide emails dos clientes
4. Use template profissional (não muito "!"


Problema: Muitos caracteres especiais?
──────────────────────────────────────
Causa: Encoding UTF-8
Solução: Automático, não pode quebrar
