📧 ABA DE MARKETING - GUIA RÁPIDO DE IMPLEMENTAÇÃO
================================================

✅ O QUE FOI INSTALADO
========================

1. ✨ NOVO: Aba "Marketing" no CRM
   - Localizção: Menu superior, entre "Pós-Venda" e "Comissões"
   - Ícone: 🚀 (envio)

2. 📦 NOVO: Pacote Nodemailer
   - npm install nodemailer@6.9.7
   - Responsável por enviar emails de verdade

3. 💾 NOVO: 3 Tabelas no Banco de Dados
   - email_campanhas: Histórico de campanhas
   - email_logs: Detalhes de cada email enviado
   - Rastreamento automático

4. 📄 NOVO: Arquivo JavaScript (marketing.js)
   - Toda lógica de funcionalidade
   - 400+ linhas de código
   - Integração com frontend

5. 🔌 NOVO: 3 Endpoints API
   - POST /api/marketing/enviar-emails
   - GET /api/marketing/historico
   - GET /api/marketing/campanha/:id


⚙️ CONFIGURAÇÃO NECESSÁRIA (IMPORTANTE!)
==========================================

Para que os emails funcionem, você PRECISA configurar credenciais SMTP.

PASSO 1: Criar arquivo .env
────────────────────────────
Crie um arquivo chamado ".env" na pasta raiz do projeto (mesmo nível de server.js):

    c:\crm-server\.env

PASSO 2: Escolha seu provedor de email
───────────────────────────────────────

🟢 OPÇÃO A: Gmail (Mais Fácil para Começar)
──────────────────────────────

a) Acesse: https://myaccount.google.com
b) Menu esquerdo: "Segurança"
c) Procure: "Senhas de app" (perto do final)
d) Selecione:
   - App: Mail
   - Device: Windows
e) Google gera uma senha de 16 caracteres
f) Copie a senha (com espaços mesmo)

g) No arquivo .env, escreva:

SMTP_HOST=smtp.gmail.com
SMTP_PORT=587
SMTP_USER=seu-email@gmail.com
SMTP_PASS=abcd efgh ijkl mnop

⚠️ Exemplo REAL:
SMTP_HOST=smtp.gmail.com
SMTP_PORT=587
SMTP_USER=joao@gmail.com
SMTP_PASS=abcd efgh ijkl mnop


🟡 OPÇÃO B: Outlook/Hotmail
────────────────────────────

SMTP_HOST=smtp-mail.outlook.com
SMTP_PORT=587
SMTP_USER=seu-email@hotmail.com
SMTP_PASS=sua-senha-outlook


🟠 OPÇÃO C: SendGrid (Conta Gratuita)
──────────────────────────────────────

a) Cadastre em: https://sendgrid.com
b) Vá a: Settings > API Keys
c) Crie uma nova chave
d) No arquivo .env:

SMTP_HOST=smtp.sendgrid.net
SMTP_PORT=587
SMTP_USER=apikey
SMTP_PASS=sua-api-key-aqui


PASSO 3: Reiniciar o Servidor
──────────────────────────────

Após criar/editar o .env:

1. Pare o servidor (Ctrl+C)
2. Rode novamente: npm start
3. Você verá uma mensagem de sucesso:
   ✅ "Servidor de email configurado com sucesso"


🚀 COMO USAR
============

1. Faça login no CRM
2. Clique na aba "🚀 Marketing"
3. Selecione clientes do lado esquerdo
4. Escreva o assunto e mensagem
5. Use {{nome}} para personalizar
6. Clique "Enviar Emails"
7. Veja o histórico na aba "Histórico"

2. Restart do servidor
3. No CRM, selecione APENAS SUA EMAIL

⚠️ AVISO: LIMITE DE EMAILS
===========================

Cada provedor tem limite:

Gmail:
  - Até 300 emails/dia (conta gratuita)
  - Até 2000 emails/dia (conta Google Workspace)

Outlook/Hotmail:
  - Até 300 emails/dia

SendGrid Gratuito:
  - Até 100 emails/dia

⛔ Se você tentar enviar mais que o limite:
   - Os primeiros vão passar
   - Os últimos vão falhar
   - Você verá os erros no histórico

🐛 TROUBLESHOOTING
===================

"Email não configurado"
⚠️ Problema: .env não foi criado/lido
✅ Solução: Crie .env, adicione credenciais, restart do servidor

"Erro ao enviar emails"
⚠️ Problema: Credenciais incorretas
✅ Solução: Verifique .env, teste com email seu

"Emails não chegam na inbox"
⚠️ Problema: Spam folder (muito comum!)
✅ Solução: Peça ao cliente verificar spam, test com seu email

"Acesso negado com Gmail"
⚠️ Problema: Usando senha do Gmail normal
✅ Solução: USE senha de app (https://myaccount.google.com/apppasswords)
